Java中的注解和反射

发表于 2025-03-11 更新于 2025-03-13

在学习Spring的时候, 经常看见在某些函数或者类上面有着@开头的字符串, 这叫做注解. 而反射在Spring中则经常和注解配合使用. 下面先聊反射:

Java中的反射 Reflection in Java

反射

所谓反射, 就是可以在程序运行时动态操作类和对象. 对于一个类, 我们可以创建它的class对象, 通过class对象来操作这个类.

获取Class对象的三种主要方式

使用类字面常量

Class<User> userClass = User.class;
可以看出这个是静态的, 在编译的时候就获得了userClass这个实例, 而且类型是很确定的Class<User>. 使用这种方式获取的时候, 不会触发类的初始化.

java中类的初始化是指为类的静态成员变量赋予初始值的过程, 以及执行静态代码块中的操作. 这个过程发生在类被装载到JVM之后, 但在任何创建该类的实例之前.

使用对象实例的getclass方法

Class<? extends User> aClass = user.getClass();
这里代码的左半边是IDEA补全的, 故严谨一些. 其实也可以写成Class<?>, 但是这样就显得宽泛了些, 这里为什么类型也可能是User的子类, 是因为编译器看到user的声明类型是User, 但是子类也是可以赋值给user的(继承多态), 故IDEA补全了Class<? extends User>.

这是getClass的签名: public final native Class<?> getClass();

使用class的forname方法

Class<?> aClass = Class.forName("org.working.User");
一般用在类名在编译时不可知的情况. 在使用这个方法的时候, 签名需要加上ClassNotFoundException异常.

注意, 使用这种方法会立即触发类的初始化.

我们还可以利用反射得到一个类的构造函数, 然后拿到一个类的实例.

这就是Class对象里面的newInstance方法.

获取到一个类的实例之后, 可以使用instanceof关键字来判断, 最后进行类型转换.

但是这不是反射的最佳实践, 因为如果编译时候就已经知道最终的类型, 不如显式调用. 反射真正的价值在于处理编译时候的未知类型, 从而编写更加具有通用性的代码.

通过反射, 私有字段或者是final字段也是可以随意修改的.

注解元素的返回类型必须是以下类型之一：

  • 基本类型（int, boolean, char, byte, short, long, float, double）
  • String
  • Class（或 Class<?>）
  • 枚举类型（enum）
  • 注解类型（即其他注解）
  • 以上类型的数组

每一个字段都可以通过default指定默认值. 注解如果只有一个元素, 标注的时候可以不需要写字段的名字, 直接@ExampleAnno("element")即可.
